02

Skills by phase

11 skills · 6 layers

Each skill is a single SKILL.md markdown file with a YAML frontmatter description that controls when Claude Code surfaces it. The skill body is the behavior contract.

Setup   one-time bootstrap

scaffold-project

setup
When Once per project. User says "set up kss", "scaffold", "bootstrap", "initialize planning". What Creates the .kss/ shell with PROJECT.md, STATE.md, CANONICAL-KB.md, and empty topic/spike/archive folders.

map-codebase

setup
When After scaffold; again whenever the codebase shape shifts meaningfully. What Generates four files under .kss/codebase/: STACK.md, STRUCTURE.md, CONVENTIONS.md (overwritten on rerun), and VOCABULARY.md (append-only — preserves curated terms).

Lifecycle   topic + milestone management

new-topic

lifecycle
When Starting an unrelated work track. User says "new topic", "switch focus to something different". What Creates .kss/topics/<slug>/ with TOPIC.md, STATE.md, LOG.md, SEEDS.md, MILESTONES.md, and an empty milestones folder. Sets the new topic active.

plan-milestone

lifecycle
When Active topic, no milestone scoped, ready to start work. User says "plan", "scope", "what should I tackle next". What Adaptive — discusses scope first if fuzzy, plans directly if not. Writes PLAN.md inside a new milestone folder. Active milestone is the one without a SUMMARY.md.

complete-milestone

lifecycle
When Milestone shipped. User says "done", "shipped", "wrap up the milestone". What Writes SUMMARY.md (which is the marker that distinguishes shipped from active), appends to MILESTONES.md, resets state. With --archive-topic, retires the entire topic to .kss/archive/.

Session   bookend every coding session

start-session

session
When Beginning of work. User says "what was I working on", "where did we leave off", "catch me up". What Reads .kss/STATE.md, the active topic's STATE.md, recent LOG entries, and the active milestone's PLAN.md. Tells you where you left off, what's next, what's blocking.

wrap-up

session
When Ending a session. User says "done for the day", "log this", "wrap up", "let's call it". What Appends a terse LOG.md entry, refreshes the topic-level STATE.md (current focus, blockers, next action), optionally writes a longer notes file under the active milestone.

Knowledge   park ideas, distill durable insights

capture

knowledge
When Mid-flow, an idea or todo surfaces. User says "remind me to", "for later", "park this", "don't forget". What Drops a seed (with a mandatory trigger condition — items without triggers rot, so capture refuses them), a backlog idea, or a scratch note. Lives in the active topic's SEEDS.md or as a standalone note.

distill

knowledge
When Notes / LOG entries piling up. Periodically — typically before complete-milestone or when the milestone notes folder has >3 files. What Reads recent work, surfaces durable insights (surprises, patterns, postmortem learnings) → CANONICAL-KB.md, plus vocabulary candidates → VOCABULARY.md. Two destinations, never crossed. Optional --prune deletes source notes after distillation.

Exploration   "what if" without polluting the milestone

spike

exploration
When Want to test or prototype an idea before committing to it. User says "let's try", "spike on", "see if it works". What Creates a throwaway folder under .kss/spikes/YYYYMMDD-<slug>/. README.md captures: question, approach, findings, verdict. Verdict is the artifact; code is incidental.

Meta   the self-improvement loop

skill-autopsy

meta opt-in
When A specific skill misfires. User says "that skill X didn't do what I wanted", "log a complaint", "review skill X reports". What Two modes. Write mode: interviews the user, appends a report to ~/kss-skill-reports/<skill>.md. Consolidate mode: reads accumulated reports, looks for recurring root causes, proposes a concrete diff to the offending SKILL.md. Diff is applied if ~/.kss-source is set, printed for manual handling otherwise. Never fabricates — "nothing to change yet" is a valid outcome.
03

The .kss/ layout

durable state

Every kss skill reads from and writes to this tree. The shape is the API.

.kss/
├── PROJECT.md             # durable identity + topics index
├── STATE.md               # project-level pointer (active topic + milestone)
├── CANONICAL-KB.md        # cross-topic distilled knowledge
├── codebase/
│   ├── STACK.md           # overwrites on map-codebase rerun
│   ├── STRUCTURE.md       # overwrites on map-codebase rerun
│   ├── CONVENTIONS.md     # overwrites on map-codebase rerun
│   └── VOCABULARY.md      # APPEND-ONLY — auto-seeded, grown by distill
├── topics/
│   └── <topic-slug>/
│       ├── TOPIC.md       # identity, success bar, key decisions
│       ├── STATE.md       # current focus, blockers (overwritten)
│       ├── LOG.md         # session-by-session, terse, newest-on-top
│       ├── SEEDS.md       # parked items, each with a trigger condition
│       ├── MILESTONES.md  # shipped milestone summaries, newest-on-top
│       └── milestones/
│           ├── <version>-<slug>/        # active (no SUMMARY.md yet)
│           │   ├── PLAN.md
│           │   └── note-YYYYMMDD-*.md
│           └── <version>-<slug>/        # shipped (has SUMMARY.md)
│               ├── PLAN.md
│               ├── SUMMARY.md
│               └── note-YYYYMMDD-*.md
├── spikes/
│   └── YYYYMMDD-<slug>/
│       └── README.md      # question, approach, findings, verdict
└── archive/
    └── <archived-topic>/  # whole topics that shipped or were abandoned

Two-level state: .kss/STATE.md is the pointer (active topic + milestone). The active topic's STATE.md is the content (what you're actually doing).

Active vs shipped: the presence of SUMMARY.md in a milestone folder is the marker.

04

The self-improvement loop, in detail

opt-in

skill-autopsy uses two filesystem locations: ~/kss-skill-reports/ for personal reports (always writable) and ~/.kss-source for the optional source-clone path (forker / contributor flow).

Reports location

~/kss-skill-reports/<skill>.md

Personal. Never committed to the plugin source. On first autopsy run, the directory is created with confirmation. Each report is one ## YYYY-MM-DD entry with: what was asked, what the skill did, why it was off, optional hypothesis.

Source path (optional)

~/.kss-source

One-line file containing the absolute path to a writable kss source folder (typically a clone of kss-plugins). When set, consolidate mode applies the proposed diff to that source. Without it, the diff is printed for manual handling.

Threshold logic

Report countBehavior
0 — 3Append silently. Patterns aren't reliable yet at this volume.
4 — 5Offer to consolidate: "we already have N reports — would you like me to explore them and identify possible improvements?"
6+Escalate: "past 5, patterns get tangled and harder to consolidate cleanly. Strongly recommend reviewing now." Caps are warnings, not refusals.

Consolidate mode — what it looks for

Signal

Recurring root causes — multiple reports pointing at the same gap.

Description mismatches — skill ran when it shouldn't have, or vice versa. Points at frontmatter description.

Missing preflight — proceeded on bad assumptions.

Instruction ambiguity — two reasonable readings produced different behavior.

Unhandled edge cases — recurring "what about when X" scenarios.

Noise

Vague reports with no shared root cause.

One-off frustrations that won't recur.

Fabricated patterns — never invents a reason. "Nothing to change yet" is the right answer when the data doesn't support a change.

On approved diff:

  • If source set: diff applied to <source>/skills/<skill>/SKILL.md, reports cleared (frontmatter preserved), commit hint printed (git -C <source> ...) — never executed.
  • If source unset: diff printed verbatim with options (fork later, open issue, adapt usage). Reports cleared on user opt-in.
05

Conventions that hold it together

8 rules

Small rules with big consequences. These are why the framework stays simple as projects grow.

RuleWhy
YAML frontmatter on every state file Easy to grep, parse, or filter by date / status programmatically.
Newest-on-top in LOG.md and MILESTONES.md When you skim, the most recent context is first. Old entries fall off naturally as they scroll.
Slugs are kebab-case Filesystem-safe, URL-safe, consistent across topics / milestones / spikes.
ISO dates: YYYY-MM-DD in frontmatter; YYYYMMDD as folder/file prefixes Sorts correctly. Folder prefixes group cleanly in ls.
SUMMARY.md presence = milestone is shipped No status field to drift. The marker is structural.
Seeds require a mandatory trigger condition Items without triggers rot. capture refuses them. Forces the user to specify when the seed becomes actionable.
Never auto-commit Skills modify files; the user runs git themselves. Keeps the framework legible — every change is a file you can review before commit.
Two-level state .kss/STATE.md is the pointer. Topic-level STATE.md is the content. Updates target the right level.
