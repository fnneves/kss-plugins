# scaffold-project

Initializes the `.kss/` planning shell at the project root. Creates the project-level files and the empty topics/spikes/archive/codebase folders. Run once per project, before any other skill.

## Pre-flight

1. **Refuse if `.kss/` already exists.**
   - Tell the user: "`.kss/` already exists at `<absolute-path>`. Refusing to overwrite. If you want to start over, archive or remove the existing `.kss/` first."
   - Exit.

2. **Optional: spec file.**
   - Ask: "Do you have a markdown file with project details? Provide an absolute path, or press Enter to describe interactively."
   - If a path is given, read it.

## Process

1. **Gather inputs.**
   - **Project name** (e.g. `predictions-repo`). Use the directory name as a default suggestion.
   - **One-line description** (what is this thing).
   - **Optional**: tech stack hint, existing constraints. (Skip if not offered.)

2. **Create the directory tree.**

   ```bash
   mkdir -p .kss/codebase .kss/topics .kss/spikes .kss/archive
   ```

3. **Write the three root files** using the templates below: `PROJECT.md`, `STATE.md`, `CANONICAL-KB.md`.

4. **Tell the user the next steps.**
   - "Scaffold complete. Next: run `map-codebase` to generate the codebase snapshot, then `new-topic` to start your first topic."

5. **Do not commit.** Let the user decide.

## Template: `.kss/PROJECT.md`

```markdown
---
name: {project-name}
created: {YYYY-MM-DD}
last_updated: {YYYY-MM-DD}
---

# {project-name}

## What This Is

{user-provided one-line description}

## Topics

(none yet — run `new-topic` to start one)

## Archived Topics

(none)

## Tech Stack

(refresh via `map-codebase`)

---
*Generated by scaffold-project on {YYYY-MM-DD}*
```

## Template: `.kss/STATE.md`

```markdown
---
active_topic: null
active_milestone: null
last_session: null
last_updated: {YYYY-MM-DD}
---

# Project State

No active topic. Run `new-topic` to start one.
```
